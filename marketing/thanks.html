Run the installer and follow the instructions

Worktale Desktop is a new open-source tool, so the installer isn’t yet signed with a commercial code-signing certificate. Chrome and Windows SmartScreen default to warning on unsigned executables from unknown publishers. The file is safe — you can verify by checking the SHA-256 against the value published in our GitHub releases once we publish it there, or by inspecting the Tauri/Rust source at github.com/worktale/worktale-cli.

Chrome — “Improve download protection” popup

  1. Click the downloads icon in the Chrome toolbar.
  2. Find worktale-desktop-v1.4.1-win-x64-setup.exe in the list.
  3. Click the … menu or arrow next to it and choose Keep (or Keep dangerous file).

Windows SmartScreen — “Windows protected your PC”

  1. When the blue SmartScreen dialog appears, click More info.
  2. Click Run anyway.
  3. The installer opens; proceed with installation.

Prefer the CLI?

You can skip the desktop app entirely and use the npm-distributed CLI — no SmartScreen warnings:

$ npm install -g worktale

A commercial code-signing certificate is being added to the install — once it ships, these warnings go away.

Once installed

Worktale Desktop reads the same local database the CLI writes to. If you already have worktale init set up on any repo, your history is already there. If not, install the CLI once:

$ npm install -g worktale
